# Time Tracking Application - Project Instructions

## Project Overview
Mobile-first time tracking web application for staffing company warehouse workers. Simple clock-in/out functionality with admin dashboard.

## Core Features
- **Worker Profiles**: Name + optional employee ID + auto-generated 4-digit PIN
- **4-Punch System**: shift_start → lunch_out → lunch_in → shift_end (sequential flow)
- **Active Shift Tracking**: Detect ongoing shifts with step tracking, prevent duplicate clock-ins
- **Admin Dashboard**: PIN-protected via server-side API (`/api/admin/verify-pin`) reading `ADMIN_PIN` env var
- **CSV Export**: Filtered export with all 4 punch columns
- **Worker Management**: Add/edit/delete workers, view/regenerate PINs
- **Trilingual i18n**: English, 中文, Español — stored in localStorage, switchable via LanguageSelector component

## Technical Architecture
- **Framework**: Next.js 14 (App Router)
- **Database**: PostgreSQL via Prisma ORM
- **Timezone**: America/New_York (Eastern Time) for all timestamps
- **Authentication**: Simple PIN protection for admin only; workers must enter their 4-digit PIN before clock-in/out
- **Worker PIN System**: Each worker has a randomly generated 4-digit PIN. PINs are auto-generated on worker creation. Admins can view/regenerate PINs from worker management page. PIN is not exposed in public API (only via `?includePin=true` param).
- **Styling**: Tailwind CSS with mobile-first approach

## Database Schema
**Worker Model**:
- id (String, UUID)
- name (String, required)
- employeeId (String, optional)
- createdAt (DateTime)

**TimeEntry Model**:
- id (String, UUID)
- workerId (String, foreign key)
- clockIn (DateTime, required)
- clockOut (DateTime, nullable)
- totalHours (Float, nullable)
- date (String, for filtering)

## Design Language
- **Colors**: Blue accent (#3b82f6) for primary actions
- **Mobile-First**: Large buttons (min 80px height), clear spacing
- **Typography**: Large readable text, minimal labels
- **Interactions**: Instant feedback, smooth animations
- **Layout**: Centered content, max-width 1200px

## Key Behaviors
- Workers don't log in - just tap their name
- Admin PIN stored in .env as ADMIN_PIN
- Active shifts detected by null clockOut
- All timestamps converted to Eastern Time
- CSV export respects current filters
- Total hours auto-calculated on clock-out

## Page Structure
- `/` - Main worker clock-in screen (large button grid)
- `/admin` - Admin dashboard with PIN protection
- `/admin/workers` - Worker management interface
- `/api/workers` - Worker CRUD operations
- `/api/time-entries` - Time entry operations
- `/api/export-csv` - Filtered CSV export
